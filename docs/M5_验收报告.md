### Live API 测试
```bash
curl -X POST http://47.99.163.232:10090/api/style/analyze \
  -d '{"userId":"test","records":[...]}'
```
**结果：❌ 超时（>10s）**

**原因分析：**
- `StyleAnalyze` handler 调用 `aiClient.AnalyzeStyle()`
- `aiClient` 向 LLM API（`https://api.openai.com/v1/chat/completions`）发起请求
- `LLM_TOKEN` 环境变量未配置或无效 → LLM API 调用超时（60s × 3次重试）
- 后端日志：`llm call failed after retries`

**结论：**
- Handler 代码 ✅ 正确
- 路由注册 ✅ 正确（路由 `/api/style/analyze` 已注册）
- LLM 配置 ❌ 未配置/不可用

**建议：**
- 配置有效的 `LLM_TOKEN` 环境变量
- 或实现 mock 模式（当 token 为空时返回预设画像）

## V04 验收：绩效指标计算正确（夏普/回撤/胜率）

### 测试方法

### 单元测试结果

验证内容：
- `WinRate`：2胜1负 → 期望 ~0.67，实测符合
- `ProfitLossRatio`：期望 ~2.5（avgWin/avgLoss），实测符合

### 代码审查结果

| 指标 | 计算方法 | 状态 |
|------|---------|------|
| **夏普比率** | `(年化收益 - 无风险利率) / 年化标准差`，无风险利率=3% | ✅ 正确 |
| **最大回撤** | `calcMaxDrawdown`：跟踪 equity peak，逐笔计算回撤率，取最大值 | ✅ 正确 |
| **胜率** | `wins / totalTrades` | ✅ 正确 |
| **盈亏比** | `avgWin / avgLoss`（avgLoss 取绝对值）| ✅ 正确 |
| **卡玛比率** | `annualReturn / maxDrawdown` | ✅ 正确 |
| **索提诺比率** | `(年化收益 - 无风险利率) / 下行标准差` | ✅ 正确 |

### 数值合理性验证

| 指标 | 合理范围 | 公式 |
|------|---------|------|
| 夏普比率 | 通常 0.5~3.0（>2 优秀）| (年化收益-3%)/年化波动率 |
| 最大回撤 | 0~1（越小越好）| (peak-equity)/peak |
| 胜率 | 0~1（一般 0.3~0.6）| 盈利交易数/总交易数 |
| 盈亏比 | 一般 1.0~3.0 | avgWin/avgLoss |

**结论：**
- 代码实现 ✅ 正确
- 单元测试 ✅ 通过
- 由于 V03 数据持久化失败，无法用真实回测数据验证 Live 指标输出
